## 🔬 核心源码逐行核心逻辑解构 (Why We Code This)

在面试或评审中，以下 3 个核心代码块直接决定了本项目的技术含金量，其底层的工程取舍逻辑如下：

### 1. 动态时间戳物理破锁线 (Anti-File-Locking)
```python
import time
pdf_target = f"structure_parser/final_victory_battle_{int(time.time())}.pdf"
```
*   **代码代表了什么**：调用系统级时间戳函数 `time.time()` 获取当前精确到微秒的总秒数，转换为整数后动态拼接为 PDF 文件名。
*   **工程价值**：在 Windows 系统下，如果文件名固定，一旦用户在后台用 WPS 或浏览器打开了 PDF 预览，操作系统就会强行启动 **文件互斥锁（File Lock）**。当 Python 下一次执行 `open(..., "wb")` 试图覆盖写入时，会被系统无情抛出 `PermissionError` 拒绝访问死锁。加入时间戳后，每次运行都会诞生一个**全宇宙独一无二的清白新文件名**，直接从物理层降维击穿、完美绕过了所有老缓存文件占用锁，保证系统 24 小时绝对不卡死。

### 2. 二维表格逆向 Markdown 重构矩阵 (Structured Parsing)
```python
tables = page.extract_tables()
for table in tables:
    markdown_table = "\n\n| " + " | ".join([str(cell) for cell in table]) + " |\n"
    markdown_table += "| " + " | ".join(["---" for _ in table]) + " |\n"
```
*   **代码代表了什么**：利用 `pdfplumber` 抓取 PDF 内部的二维数据列表（List of Lists），并用 Python 标志性的 `" | ".join(...)` 语法，在格子的两头和中间强行用竖线 `|` 穿起来，在表头下方自动补上由 `---` 组成的减速带。
*   **工程价值**：传统的读取器在处理表格时，会把数字**无脑横着拉成一长串没有空格的死字符串面条**（如将实验组 99 分、对照组 12 分连成 `9912`），导致大模型语义断裂发生严重幻觉。本段代码在内存里**纯手工把碎渣数字重构为了大模型最爱看的标准 Markdown 框线布局**。大模型扫描到这些竖线，能毫秒级看清行列对比关系，**彻底物理封杀了数字幻觉**。

### 3. ReportLab 尺寸死锁与跨平台防御 (Zero-Collapse Engine)
```python
t = Table(table_data, colWidths=[150, 100, 100])
```
*   **代码代表了什么**：显式硬编码锁死 Table 控件的横向像素宽度（第一列占 150 像素，第二三列各占 100 像素）。
*   **工程价值**：如果不写 `colWidths` 参数，ReportLab 引擎会在编译的最后一毫秒（`_endBuild` 阶段），偷偷调用底层 C 语言函数去瞎猜每个单词有多宽。因为本项目为了 100% 跨平台兼容、强行屏蔽了本地的所有系统字体路径，一旦引擎找不到字体的度量衡数据，它算出来的宽度会直接变成 0 或负数，在落笔画图时会引发**零除溢出或坐标越界崩溃**。显式传入安全列宽，就是为了**直接用绝对尺寸焊死几何坐标，封杀所有由于系统字体缺失带来的崩溃隐患**。
